Configure and customize a parameter of the connector descriptor of the connector.

== Set a Display Name for a Parameter

To set a display name for a parameter, you must use the
`displayName` property, which is located in the parameter node.

You can set a display name for any parameter type. Define parameters using the
`queryParameter`, `uriParameter`, or `header` nodes. The parameters refer to the corresponding
parameters defined in the API specification.

[source,yaml]
----
endpoints:
   /content_search:
      operations:
         get:
            pagination: content-search-pagination
            displayName: Get Content Search
            description: Provides keyword and refinement search functionality for content assets.
            expects:
               queryParameter:
                  count:
                     displayName: count
                     description: The maximum number of instances per request.
----

The name you set for the parameter appears in Anypoint Studio.

image::parameter-details-studio.png[]

== Set a Description for a Parameter

To set a description for a parameter, you must use the `description`
property, which is located in the parameter node.

You can set a description for any parameter type. Define parameters using the
`queryParameter`, `uriParameter`, or `header` nodes. The parameters refer to the corresponding
parameters defined in the API specification.

[source,yaml]
----
endpoints:
   /content_search:
      operations:
         get:
            pagination: content-search-pagination
            displayName: Get Content Search
            description: Provides keyword and refinement search functionality for content assets.
            expects:
               queryParameter:
                  count:
                     displayName: count
                     description: The maximum number of instances per request. Default value is 25.
----

Set the `description` property to give the user some context or an explanation
of what the parameter does. The description you set for the parameter appears
in Anypoint Studio as a tooltip beside the parameter.

image::parameter-description-tooltip.png[]

== Ignore a Parameter

To ignore a parameter, you must use the `ignored` property, which is located in
the parameter node.

You can ignore a parameter for any parameter type. Parameters can be ignored when
defined in both operations and security schemes. When a parameter is ignored in
the connector descriptor, the parameter is not present in the generated connector.

[source,yaml]
----
security:
 oauth2:
   headers:
     Authorization:
       ignored: true

endpoints:
   /content_search:
      operations:
         get:
            expects:
               queryParameter:
                  count:
                     ignored: true
----

The `Authorization` header is ignored from the `oauth2` security definition
as the header is automatically generated by REST SDK when there is an OAuth scheme
defined. The example also shows how to ignore a query parameter from an operation.

== Select a Query Parameter Array Format

To select a query parameter array format, you must use the `queryParameterArrayFormat`
property, which can be used globally at the root of the connector descriptor or
in the operation node.

Query parameter arrays are handled in different ways in different REST
API implementations. Currently, REST SDK supports two query parameter array formats:

* `comma`
+
Sends the array values using the same key and splits the array values with commas, for example,
`?key=value1,value2,value3`

* `multimap` (default)
+
Sends the array values by repeating the query parameter key for each value in the query string, for example,
`?key=value1&key=value2&key=value3`

REST SDK uses the multimap format by default if you do not add any
information to the connector descriptor.

[source,yaml]
----
queryParameterArrayFormat: multimap

endpoints:
 /operation/{names}:
   operations:
     get:
       queryParameterArrayFormat: comma
----

In this example, the `GET /operation/\{names}` operation uses the `comma`
query parameter array format, but all of the other operations defined in
the connector use the `multimap` query parameter array format.
